# Excalidraw GitHub 编辑器 - Vibe Coding 实验项目

<!-- Vercel 部署按钮已移除 -->

本项目是一个实验性的 Web 应用，允许用户浏览 GitHub 仓库，直接在浏览器中使用 Excalidraw 编辑器打开 `.excalidraw` 文件，进行修改，并通过提交信息将更改保存回 GitHub。

**本项目的主要意义不仅在于其功能，更在于其创建过程：它完全通过 Vibe Coding 开发，利用 AI 编码助手（如 Claude、带工具的 ChatGPT 或类似工具）完成，没有人工编写代码。** 这是对 AI 驱动的软件开发工作流程的一次演示和探索。

## 核心功能

*   **GitHub 仓库浏览**: 使用具有 `repo` 范围权限的个人访问令牌 (PAT) 连接到你的 GitHub 帐户。
*   **仓库和分支选择**: 从你的帐户中选择仓库和分支。
*   **文件树导航**: 查看所选分支的文件结构。
*   **Excalidraw 文件编辑**: 点击 `.excalidraw` 文件，在嵌入式 Excalidraw 编辑器中打开它们。
*   **智能路由系统**: 
    *   自动生成可分享的URL，包含仓库、分支和文件信息
    *   支持页面刷新后恢复编辑状态
    *   URL格式：`/repo/{仓库名}/branch/{分支名}/file/{文件路径}`
    *   支持浏览器前进后退导航
*   **智能本地缓存**: 
    *   自动缓存未提交的文件更改到浏览器的 IndexedDB
    *   即使切换文件或刷新页面，未保存的更改也不会丢失
    *   支持按仓库和分支管理缓存
    *   提供缓存管理器界面，可查看、恢复和清理缓存文件
*   **变更检测**: 基于内容比较，在文件树中用星号 (`*`) 标记已修改的文件。
*   **保存到 GitHub**: 使用自定义提交信息将更改保存回 GitHub 仓库。
*   **文件操作**: 创建新文件，重命名/删除现有文件（对文件夹操作提供指引）。
*   **国际化 (i18n)**: 支持中英文界面语言，自动检测浏览器偏好。
*   **(计划中)** 查看文件提交历史。

## 本地缓存功能详解

### 自动缓存机制
- **实时保存**: 在编辑 Excalidraw 文件时，系统会自动将更改保存到本地缓存（防抖1秒）
- **智能恢复**: 重新打开文件时，如果存在缓存版本，会优先加载缓存内容
- **状态保持**: 缓存包含完整的绘图元素和应用状态，确保编辑体验的连续性

### 缓存管理器
点击工具栏中的数据库图标 (📊) 打开缓存管理器，可以：
- **查看缓存**: 按仓库和分支分组显示所有缓存文件
- **恢复文件**: 一键恢复缓存的文件内容到编辑器
- **清理缓存**: 支持清除当前分支缓存或所有缓存
- **时间信息**: 显示每个缓存文件的最后修改时间

### 缓存策略
- **自动清理**: 文件成功保存到 GitHub 后，对应的缓存会自动清除
- **分支隔离**: 不同分支的缓存相互独立，切换分支时会清理当前分支缓存
- **容量管理**: 基于 IndexedDB，支持大容量本地存储
- **智能保护**: 当切换文件时如有未保存更改，会提示用户选择：
  - **保存**: 将更改提交到 GitHub 并清除缓存
  - **放弃**: 放弃本次保存操作，但保留本地缓存（下次打开仍可恢复编辑状态）
  - **取消**: 取消切换操作，继续编辑当前文件

## 智能路由系统详解

### URL结构设计
应用采用语义化的URL结构，便于理解和分享：
- **首页**: `/` - 应用主页
- **仓库页**: `/repo/{仓库名}` - 显示指定仓库
- **分支页**: `/repo/{仓库名}/branch/{分支名}` - 显示指定仓库的指定分支
- **文件页**: `/repo/{仓库名}/branch/{分支名}/file/{文件路径}` - 打开指定文件进行编辑

### 路由功能特性
- **状态同步**: URL与应用状态实时同步，切换仓库、分支或文件时自动更新URL
- **深度链接**: 支持直接访问特定文件的URL，页面加载后自动恢复到对应状态
- **刷新恢复**: 页面刷新后根据URL自动恢复仓库、分支和文件选择状态
- **浏览器导航**: 支持浏览器的前进、后退按钮进行导航
- **URL编码**: 自动处理特殊字符的URL编码，确保路径安全

### 使用场景
- **协作分享**: 将特定文件的URL分享给团队成员，直接打开对应文件
- **书签管理**: 为常用文件创建浏览器书签，快速访问
- **工作流集成**: 在其他工具中嵌入文件编辑链接
- **状态恢复**: 意外关闭页面后，通过浏览器历史快速恢复工作状态

## 快速开始

1.  **克隆仓库:**
    ```bash
    git clone https://github.com/icattlecoder/excalidraw-gh.git
    cd excalidraw-gh
    ```
2.  **安装依赖:**
    ```bash
    npm install
    ```
3.  **生成 GitHub 个人访问令牌 (PAT):**
    *   前往你的 GitHub [开发者设置](https://github.com/settings/tokens)。
    *   生成一个新的 **classic** 令牌。
    *   给它起个名字 (例如, "Excalidraw Editor")。
    *   选择 `repo` 范围权限 (完全控制私有仓库)。
    *   复制生成的令牌。**你将无法再次看到它。**
4.  **运行开发服务器:**
    ```bash
    npm run dev
    ```
5.  在浏览器中打开指定的本地 URL (通常是 `http://localhost:5173`)。
6.  在提示时粘贴你的 GitHub PAT。该令牌存储在你浏览器的 IndexedDB 中，不会发送到除 GitHub API 之外的任何地方。
